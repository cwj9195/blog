  • 对于 JavaScript/TypeScript 项目：这个文件用于配置 JavaScript 或 TypeScript 项目的编译选项、语言服务设置等。它可以帮助提供更好的编辑器支持，例如代码补全、语法检查、跳转到定义等功能。

    • jsconfig.json 是为 JavaScript 项目提供的配置文件,专门为 JavaScript 项目设计的。
      • 编译: 不会触发 TypeScript 编译器进行 JavaScript 文件的编译，但可以用于配置 VS Code 的智能感知。
      • 用途: 主要用于配置 VS Code 中 JavaScript 项目的代码智能感知和导航。
    • tsconfig.json 是为 TypeScript 项目提供的配置文件,是专门为 TypeScript 项目设计的。
      • 编译: 会触发 TypeScript 编译器对 TypeScript 文件进行编译。
      • 用途: 主要用于配置 TypeScript 项目的编译选项和代码智能感知，同时也可以用于配置 VS Code 中 TypeScript 项目的导航和类型检查。
      • 为项目提供类型检查和智能感知：VS Code 和其他支持 TypeScript 的 IDE 会读取这个文件来提供开发体验。

  • TypeScript的配置文件,那ts是干嘛的？类型提示 类型编译的。所以tsconfig.json是给ts编译用 给编译器提示

tsconfig.json 的顶层字段有： ​

  • compileOnSave 行为开关

  • typeAcquisition 本项目中的自动类型检查，2.1 以上支持

  • extends 继承一份配置

  • exclude 排除一些文件

  • references 项目引用，配置可以切成多块来分别设置

  • files 文件列表

  • include 指定一个或多个路径模式，告诉 TypeScript 编译器要包含哪些文件或文件夹进行编译和类型检查。

    • 一定得加上 .vue，不加得话ts编译器不会去编译.vue的文件，在导入.vue文件时就会没有类型提示
    • 如果没有 include 字段，TypeScript 默认会从包含 tsconfig.json 文件的目录开始递归地包含所有的 .ts、.tsx 和 .d.ts 文件。

tsconfig.json 和 vite.config.ts 里路径别名的区别 ​

大多数场景下，可以这样简单理解：编辑器中ts的校验提示，是用tsconfig中的配置；和项目的打包编译相关的，是用vite.config。

  • 其实就是解耦的缘故。比如你可能压根不用 TS 去开发、只是单纯的 JS；而 tsconfig.json 也不会去考虑你是不是用的 vite。二者在这点上是没有耦合的。
    • vite.config.js才能保证运行没有错误 resolve.alias是给 Rollup 看的，实际是 @rollup/plugin-alias 这个插件的配置项，用于 Rollup 构建时模块化查找路径。
    • tsconfig.json 里的是给 tsc 看的，用于 TypeScript 构建时模块化查找路径。对与程序运行是没有影响的 别名只能保证编译没有错误

  • 基本类型：string number boolean symbol null undefined bigint

  • 引用类型：array Tuple(元组) object(包含Object和{}) function

    • array
    ts
      // 在元素类型后面加上[]
  let arr: number[] = [1, 2];

  // 或者使用数组泛型
  let arr: Array<number> = [1, 2];

    • Tuple 元组

      • 元组类型用来表示已知元素数量和类型的数组，各元素的类型不必相同，对应位置的类型需要相同。

    • object

      • object 非原始类型，在定义上直接使用 object 是可以的，但你要更改对象的属性就会报错，原因是并没有使对象的内部具体的属性做限制，所以需要使用 {} 来定义内部类型

      • Object(大写的O）,代表所有的原始类型或非原始类型都可以进行赋值,除了null和`undefined

  • 特殊类型：any unknow void nerver Enum(枚举)

    • any 在 TS 中，任何类型都可以归于 any 类型，所以any类型也就成了所有类型的顶级类型

      • 比如我们可以把任何类型的值赋值给 any 类型的变量，也可以把 any 类型的值赋值给任意类型（除 never 以外）的变量

    • unknow 与any一样，都可以作为所有类型的顶级类型，但 unknow更加严格，那么可以说除了any 之下的第二大类型，接下来对比下any,主要严格于一下两点：

      • unknow会对值进行检测，而类型any不会做检测操作，说白了，any类型可以赋值给任何类型，但unknow只能赋值给unknow类型和any类型
      • unknow不允许定义的值有任何操作（如 方法，new等），但any可以
      • 与 any 不同的是，unknown 在类型上更安全。比如我们可以将任意类型的值赋值给 unknown，但 unknown 类型的值只能赋值给 unknown 或 any

    • void 当一个函数，没有返回值时，TS会默认他的返回值为 void 类型

    • nerver 表示一个函数永远不存在返回值，TS会认为类型为 never，那么与 void 相比, never应该是 void子集， 因为 void实际上的返回值为 undefined，而 never 连 undefined也不行

      • 符合never的情况有：当抛出异常的情况和无限死循环

interface ​

  • 使用interface关键字来定义接口

  • 接口可以用来描述对象，主要可以包括以下数据：可读属性、只读属性、任意属性

    • 可读属性：当我们定义一个接口时，我们的属性可能不需要全都要，这是就需要 ? 来解决
    • 只读属性：用 readonly修饰的属性为只读属性，意思是指允许定义，不允许之后进行更改
    • 任意属性：这个属性极为重要，它是可以用作就算没有定义，也可以使用，比如 [data: string]: any。比如说我们对组件进行封装，而封装的那个组件并没有导出对应的类型，然而又想让他不报错，这时就可以使用任意属性
    • 在多个相同名称的 interface 中同名的 interface 声明会被自动合并
    ts
      interface Props {
      a: string;
      b: number;
      c: boolean;
      d?: number; // 可选属性
      readonly e: string; //只读属性
      [f: string]: any //任意属性
  }
  let res: Props = {
      a: '小杜杜',
      b: 7,
      c: true,
      e: 'Domesy',
      d: 1, // 有没有d都可以
      h: 2 // 任意属性，之前为定义过h
  }

  res.e = 'hi' // error, 原因是可读属性不允许更改

typeof ​

语法：typeof [值 | 值的变量]

  1. js的typeof是值运算
  2. ts的typeof是类型运算
ts
// ts的类型运算
let a = 1;
let b:typeof a; 
// `let b:typeof a;` ==========等价========= `let b:number;`  
// `typeof a` 类型运算结果是number

// `let b: typeof a | '1' ` ==========等价========= `let b:number | '1';`   
//变量b的类型是 原始类型number 或者 值类型的字符串'1'

// js的值运算
if (typeof a === 'number') {  b = a; }
ts
type T = string | number | 'string';  //正确
// typeof Date()=='string'
type T = string | number | typeof Date();   //错误  由于编译时不会进行 JavaScript 的值运算，所以TypeScript 规定，typeof 的参数只能是标识符，不能是需要运算的表达式。

in keyof 映射类型 ​

  • in 运算符在TypeScript中用于迭代联合类型中的每个成员类型。当你有一个联合类型 U，in 运算符可以用来遍历 U 中的每个成员类型，并为每个成员类型执行某些操作。

  • keyof 运算符用于从对象类型中提取所有键的联合类型 包含了对象类型中所有属性的名称。

ts
import type { ButtonProps } from 'ant-design-vue/es/button'
type button = {[i in keyof ButtonProps]: ButtonProps[i]}

  • 注意：映射类型主要用于定义一个类型，该类型是基于另一个类型中的键值对进行映射的。

  • 映射类型本身不应该直接声明属性或方法，而是应该用于生成其他类型的结构。

  • 如果你需要在映射类型中声明属性,可以换种方式,先定义一个接口或类型别名 使用继承extends或交叉&

cclass Car {
    // 只读属性，仅可在构造函数中修改
    readonly r = '只读属性'
    // 只读属性，仅可在构造函数中修改
    readonly rf = () => '只读方法'
    // 私有属性
    private p: string = '我是私有属性，只能类自身内部访问'
    // 私有方法
    private pf = () => '我是私有方法，只能类自身内部调用'
    // 受保护的属性
    protected pd: string = `受保护的属性，自身内部和子类中 可以访问，外部无法访问`
    // 受保护的方法
    protected pdf = () => `受保护的方法，自身内部和子类中 可以访问，外部无法访问`

    // 静态属性
    static x: string = '静态属性，只有类可以访问 car.x';
    // 静态方法
    static f() { return '静态方法，只有类可以调用 car.x()' };
    // 字段 默认public,隐藏显示了
    public fields: string;
    fields1: number | undefined = 1;
    fields2: string | undefined = '1';
    // 方法 默认public,隐藏显示了
    disp(): void {
        console.log("函数中显示发动机型号  :   " + this.fields)
    }
    public disp1() { }
    // 构造函数
    constructor(fields: string, fields1?: number, fields2?: string) {
        this.fields = fields
        this.fields1 = fields1
        this.fields2 = fields2
        console.log(this.p)
        console.log(this.pf())
    }

}
