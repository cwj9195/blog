ESM、CJS、AMD ​

  1. ESM（ES Modules） 的模块化

    1. export && import
    2. 编译时加载 就能确定模块的依赖关系以及输入和输出的变量(按需加载)
    3. 效率要比 CommonJS 模块的加载方式高
    4. 一个文件里，有且只能有一个 export default。但可以有多个 export。
    5. 如果导入的是文件夹 他会找 目录下的 index.js 文件
    6. tips ES2020 import()函数 区别于 import 命令 前者是运行时加载的(所以他可以使用条件语句来按需执行)返回 promise 对象
    js
    export Amoy;
import Amoy from "./Amoy.js";


export {Amoy:Amoy};
import {Amoy} from "./Amoy.js";


export default "Amoy";  //export default XX 相当于将'引用或值'赋值给 default 然后将整个 Module 导出
export default {Amoy};
import Amoy from "./Amoy.js";


// tips:如果直接import '地址',不导入模块里的任何内容,只执行
import "******"; //执行 不导入

  2. CJS（CommonJS） (社区)

    1. module.exports && require
    2. 运行时加载 才能确定模块的依赖关系以及输入和输出的变量(整体加载)
    3. CommonJS 是为了 server 服务端设计的 Nodejs 就是 CommonJS 规范的实现者
    4. 它是一个**同步加载模块的模式**
    5. CommonJS 模块的加载机制是，输入的是被输出的值的拷贝。也就是说，一旦输出一个值，模块内部的变化就影响不到这个值。
    6. 可兼容 AMD 模块的导出
    7. require
    8. 如果有路径，就去路径里面找
    9. 没有的话就去 node_modules 里面找
    10. 再去 node 的安装目录里找
    js
    // module.exports 只导出（对外暴露）一个静态方法
module.exports = 1;

// exports可多个
exports.a = 1;
exports.b = 2;

const m = require();

// module.exports 只能一个
module.exports = "Amoy" || { a: 1, b: 2 };
const m = require("./content.js");

  3. AMD（Asynchronous Module Definition）(社区)

    1. 运行时加载 才能确定模块的依赖关系以及输入和输出的变量(整体加载)
    2. define && require （前者用来定义模块，后者用来调用模块）
    3. AMD 是为了 浏览器设计的 RequrieJS 就是 AMD 规范的实现者
    4. 它是一个可**异步加载模块的模式**
    js
      <!-- RequrieJS的导入导出用的是方法不是命令 -->
  define('content.js', function(){
    return 'Amoy';
  })
  require(['./content.js'], function(animal){
    console.log(animal);
  })

class 类 ​

  1. 单例模式
    1. 有多个实例 这个构造函数只实例化一次 通过静态方法返回实例
    2. Person.staticFuc()
    3. 单例模式的从实现步骤上来讲，分为三步：
      1. 构造方法私有，保证无法从外部通过 new 的方式创建对象。
        1. module.exports 只导出（对外暴露）一个静态方法
        2. return 的就是内部 new 出来的实例 return new 类名（） 而不是类
      2. 对外提供获取该类实例的静态方法
      3. 类的内部，创建该类的对象，通过第 2 步的静态方法返回
      4. 这种单例模式需要当前模块只暴露这一个静态方法 外部共享这一个实例
  2. 实例化调用

    构造函数里的属性方法以及原型链上的属性方法都属于这个实例方法

    1. 构造函数里的属性方法
    2. 原型链上的属性方法 prototype
  3. 继承
    1. ES5 的继承使用 对象冒充结合原型链一起来实现
  • tips:
    1. 声明类里面 方法之间不用逗号

class Person {
  //获取实例
  static getInstance(param) {
    // 静态方法这里找不到this
    if (!Person.instance) {
      Person.instance = new Person();
    }
    return Person.instance;
  }
  // 类构造函数 new实例化的时候执行这个constructor()
  constructor(name, age) {
    this._name = name;
    this._age = age;
  }
  //实例方法
  getName() {
    console.log(this._name);
  }
  setName(name) {
    this._name = name;
  }
  // 静态方法
  static fun2() {
    // 静态方法这里找不到this
  }
}
// 静态方法
Person.setName = function () {
  console.log("这是静态方法");
};
// 原型链上的属性和方法
Person.prototype.sex = "男";
Person.prototype.work = function () {
  console.log(`${this.name}---${this.age}--${this.sex}`);
};

// 执行静态方法
Person.setName();
// 实例方法是通过实例化来调用的，静态方法是通过类名来调用的

p.run();
p.work();

编译时 || 运行时 ​

  1. 编译时错误
    • 编译时的错误一般指源代码中存在的语法错误，类型检查错误，编译器崩溃（极少发生），通常编译器会指出是源代码中的哪一行出现了错误，编译成功的程序才有可能被运行。
  2. 运行时错误
    • 编译成功的程序也可能在运行的过程中产生错误，可能是某些功能不好用，或者是一些常见的运行时错误比如除零，引用空指针，OOM 等，这些错误被称为运行时错误。

apply、call、bind ​

  • 在 javascript 中，call 和 apply 都是为了改变某个函数运行时的上下文（context）而存在的， 换句话说，就是为了改变函数体内部 this 的指向。 JavaScript 的一大特点是，函数存在「定义时上下文」和「运行时上下文」以及「上下文是可以改变的」这样的概念。
  1. apply(this,[参数,参数])
  2. call(this,参数,参数)
  3. 注意： call()方法的作用和 apply() 方法类似，区别就是 call()方法接受的是参数列表，而 apply()方法接受的是一个参数数组。
